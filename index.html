What I Learned

For the startup section, I experimented with CSS Grid and subgrid layouts:

The display: grid property creates a grid container, while grid-template-columns controls the number and size of columns. I found that using repeat(3, 1fr) creates three equal-width columns that adjust responsively.

I discovered how to use distinct visual styling for different card types by creating modifier classes (like .website-card, .ios-card, etc.) that apply different background colors and border accents.

For responsive design, I learned how to use media queries to adjust the grid layout on smaller screens, switching to a single column with grid-template-columns: 1fr when the viewport width is below 768px.

I also implemented hover effects with transform: translateY(-5px) and enhanced box-shadow to create an interactive "lifting" effect when users hover over cards.

What I Learned

I explored two HTML elements I hadn't used before: the <video> element and the <details> element (which you're seeing right now!).

The <video> elementprovides native support for playing videos without requiring third-party plugins. I learned that it supports multiple source formats for browser compatibility, allows for custom controls, and can be styled with CSS to fit different layouts. One interesting behavior I discovered is that the poster attribute lets you display a thumbnail before the video plays.

The <details> element (which you just clicked on, also the ones in the previous categories) creates an interactive disclosure widget that reveals additional information when toggled. It includes a <summary> element that serves as the visible heading!
